# Approval Gate

Do not implement UI code until every item is true.

- [ ] Product context has been summarized from repository and user input.
- [ ] At least 2 style options were presented with explicit tradeoffs.
- [ ] Palette and typography were discussed and user-confirmed.
- [ ] Motion and accessibility constraints were confirmed.
- [ ] Scope boundaries (what is in/out) were agreed.
- [ ] User explicitly approved one option or a defined hybrid.
- [ ] Final brief is restated in chat before coding starts.

If any checkbox is false, continue discussion and do not modify implementation files.
