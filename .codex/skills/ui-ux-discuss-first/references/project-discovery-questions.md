# Project Discovery Questions

Ask in rounds. Start with Round 1, then continue only where answers are incomplete.

## Round 1: Product Context

1. What is the core product goal for this UI?
2. Who are the primary users and what is their level of design/tech fluency?
3. What action must this page make easier (buy, sign up, compare, monitor, etc.)?
4. What should users feel first: trust, speed, premium, playfulness, authority?
5. Which 1-2 references feel close to the desired direction?

## Round 2: Visual Direction

1. Should this UI feel minimal, expressive, editorial, data-dense, or bold experimental?
2. Should the base be light, dark, or both from day one?
3. Which color families are preferred and which must be avoided?
4. What typography tone is preferred (neutral, technical, humanist, luxury, playful)?
5. How much motion is acceptable: low, medium, or high?

## Round 3: Constraints and Scope

1. Which pages/components are in scope right now?
2. Which framework/design system constraints must be respected?
3. Any accessibility non-negotiables (WCAG level, reduced motion, contrast targets)?
4. Are there delivery deadlines that require simpler visual decisions?
5. What is explicitly out of scope for this iteration?
